6. Event Handling

๐ฏ Learning Objectives:

  • Understand how events work in JavaScript
  • Learn to handle different types of user interactions
  • Master event listeners and event objects
  • Practice preventing default behaviors

๐ Key Concepts:

Events are actions that happen in the browser:

  • Mouse Events: click, hover, drag, scroll
  • Keyboard Events: keydown, keyup, keypress
  • Form Events: submit, change, focus, blur
  • Window Events: load, resize, scroll

Event Handling:

  • addEventListener() - Attach event handlers
  • removeEventListener() - Remove event handlers
  • preventDefault() - Stop default behavior
  • stopPropagation() - Stop event bubbling
// EVENT HANDLING IN JAVASCRIPT
console.log("=== EVENT HANDLING DEMO ===");

        this.style.transform = 'scale(1.05)';

    });
    
    // Mouse leave event
    hoverDiv.addEventListener('mouseleave', function() {
        this.style.backgroundColor = 'lightgray';
        this.style.transform = 'scale(1)';
        console.log('Mouse left the div');
    });
    
    // Click event
    hoverDiv.addEventListener('click', function(event) {
        console.log('Div was clicked!');
        console.log('Click coordinates:', event.clientX, event.clientY);
        
        // Change text on click
        this.textContent = 'Clicked! Click again to reset.';
        this.style.color = 'white';
    });
    
    // Double click event
    hoverDiv.addEventListener('dblclick', function() {
        this.textContent = 'Hover over me!';

๐ก Code Explanation:

Event Listeners: Use addEventListener() to attach event handlers. This method allows multiple handlers for the same event.

Event Object: Every event handler receives an event object with useful properties like target, type, and preventDefault().

Event Delegation: Attach listeners to parent elements to handle events from child elements. This is efficient for dynamic content.

Preventing Default: Use preventDefault() to stop the browser's default behavior (like form submission or link navigation).

Custom Events: Create and dispatch custom events for communication between different parts of your application.

Best Practice: Always remove event listeners when elements are removed to prevent memory leaks.

7. Promises and Async/Await

Working with asynchronous JavaScript.

// Promise
function fetchData() {
    return new Promise((resolve, reject) => {
        setTimeout(() => {
            resolve("Data fetched successfully!");
        }, 2000);
    });
}

// Async/Await
async function getData() {
    try {
        console.log("Fetching data...");
        const result = await fetchData();
        console.log(result);
        return result;
    } catch (error) {
        console.error("Error:", error);
    }
}

// Fetch API
async function fetchUserData() {
    try {
        const response = await fetch('https://jsonplaceholder.typicode.com/users/1');
        const user = await response.json();
        console.log("User:", user.name);
    } catch (error) {
        console.error("Fetch error:", error);
    }
}
